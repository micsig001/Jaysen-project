# 演示脚本 — 5 分钟客户/老板走查

> 适用场景：销售/售前面向客户演示、PMO 向领导汇报、技术评审现场走查。
> 总时长：**5 分钟**（4 段剧情 + 1 段答疑），每段约 60–90 秒，可按需压缩/扩展。
> 推荐浏览器：Chrome 最新版，分辨率 1440×900 或以上，预先登录管理员账号。

---

## 🎯 开场（30 秒）

**对着客户说：**

> "各位好，今天我们演示 **企业级任务协同管理系统**。简单说，就是把企业微信里散落的口头任务，转成系统里可追踪、可审计、可统计的工单。整套系统 **零密码登录**，员工在企微工作台点一下就进。"

**操作：**

1. 打开企业微信 → 工作台 → 点击"任务协同"应用。
2. 自动跳转回管理后台首页（不需要输密码）。

**卖点强调：**

- 🔑 **零密码** —— 没有忘记密码、IT 重置密码的运维负担。
- 🚀 **零学习成本** —— 员工不需要安装新 App，不需要培训。

---

## 🎬 第一幕：发起一个任务（60 秒）

**剧情：** 经理小王要给员工小李派一个"季度报表分析"任务。

**操作步骤：**

1. 顶部菜单 → **新建任务**。
2. 填写：
   - 标题：`季度报表数据分析`
   - 描述：`完成 Q2 销售数据的同比/环比分析，输出 PPT 给管理层`
   - 优先级：**高**
   - 执行方 UserID：`lihua`（演示前先在管理后台查到 UserID）
   - 预计耗时：`240 分钟`
3. 点击 **创建任务**。

**系统自动：**

- 生成全局唯一的 `taskNo`（屏幕高亮这一栏 2 秒）。
- 推送企微消息给执行方（手机演示此步时切换到企微 App）。
- 数据库写入审计日志：`CREATED by wang@company.com`。

**话术：**

> "大家看，经理这边一提交，**执行方的企业微信立刻就收到通知**。从口头派活到系统派活，闭环就形成了。"

---

## 🎬 第二幕：双重确认状态机（90 秒）

**剧情：** 执行方小李收到任务 → 确认接收 → 工作中 → 提交 → 经理验收。

**操作步骤：**

1. 切到执行方账号（演示前提前登录，或用浏览器隐身窗口）。
2. 进入 **我的任务** → 看到刚才那个任务。
3. 点击 **确认接收** → 弹窗确认 → 状态变成 `进行中`。
4. 在 **任务详情** 页：
   - 顶部状态机图示高亮 `IN_PROGRESS`（UI 已显示）。
   - "实际开始时间" 自动填入当前时间。
5. 点击 **提交** → 输入备注：`已完成初步分析，请验收` → 提交。
6. 切回经理账号 → 看到状态变成 `待验收`。
7. 点击 **验收通过** → 任务状态变成 `已完成`。

**话术：**

> "注意看，这里有一个**双重确认**的设计：执行方先要"接收"——不是被动接受，而是主动承诺；做完之后，**发起方还要验收**。这就把传统 IM 里的口头活，变成了有签收记录的工单。扯皮的空间被系统掐死了。"

**卖点强调：**

- 📜 **签收证据** —— `actualStartTime` / `completedAt` 都有时间戳，可用于绩效考核。
- 🔄 **可驳回** —— 如果验收不通过，可以驳回重新做（演示时可省略这一步，文字提及）。

## 🎬 第三幕：关系图谱（90 秒）

**剧情：** 老板想看"销售部今年谁跟谁合作最多"。

**操作步骤：**

1. 顶部菜单 → **关系图谱**（仅 PC 端可见，移动端会自动屏蔽）。
2. 选择 **多人全景图** 模式。
3. 输入：`sales_lead1, sales_lead2, sales_lead3, sales_lead4`（演示前先准备 4 个销售人员的 UserID）。
4. 点击 **生成全景图**。
5. 屏幕上出现 ECharts 力导向图，节点是人，连线粗细代表合作任务数。
6. 鼠标拖动节点，悬浮查看某条边的合作任务数。

**话术：**

> "这是关系图谱。我们看到这 4 个销售今年的合作关系——**边的粗细代表协作任务数**。左边这位 sales_lead1 跟另外三人都深度合作，是个 hub 节点；sales_lead2 跟 sales_lead4 几乎没往来——这可能是团队协作的盲区。"

**卖点强调：**

- 🕸️ **看见组织** —— 不再凭印象判断谁跟谁合作。
- 🚨 **识别孤岛** —— 自动发现协作不足的人员。
- 📈 **量化协作** —— 任务数就是最硬的指标。

## 🎬 第四幕：权限与审计（60 秒）

**剧情：** 演示管理员后台 + 数据脱敏。

**操作步骤：**

1. 用管理员账号登录。
2. 进入 **管理后台** → 顶部看到三种角色的统计卡片（普通员工 / 部门经理 / 超级管理员）。
3. 进入 **用户列表** → 点击某员工 → 修改角色。
4. 切到普通员工账号 → 看自己手机号显示为 `138****5678`（脱敏）。
5. 点击手机号 → 二次确认弹窗 → 60 秒可见明文 → 自动隐藏。

**话术：**

> "权限分三级：普通员工只能看自己部门的数据，部门经理看整个部门，超级管理员看全公司。而且所有敏感字段——手机号、身份证、薪资——**默认脱敏**，要看明文需要二次确认，并且 **60 秒后自动失效**。"

**卖点强调：**

- 🔐 **三级权限** —— 数据隔离有保障。
- 🎭 **脱敏 + 审计** —— 既保护隐私，又能追溯谁看了谁的明文（后台有审计日志）。

## 🎤 答疑环节（30 秒 + 提问）

**常见客户问题与快速应答：**

| 问题 | 30 秒应答 |
|------|----------|
| **数据存哪？安全吗？** | MySQL 主存储，Redis 缓存；JWT 加密 Token，HTTPS 传输；数据库密码、企微 secret 都走环境变量，不入代码。 |
| **能跟钉钉/飞书集成吗？** | 当前是企微原生对接。架构上认证层是抽象的，加一个 `DingTalkAuthService` 即可，1-2 周工作量。 |
| **二次开发容易吗？** | 后端 Spring Boot + MyBatis-Plus，单表 CRUD 不用写 SQL；前端 Vue 3 + Element Plus，业务组件已经齐全。 |
| **价格？** | 见商务方案（不在技术演示范围）。 |

更完整的问题清单见 [FAQ.md](FAQ.md)。

## 📋 演示前 Checklist

> 演示前 30 分钟务必完成以下准备：

- [ ] 后端服务已启动（`docker-compose up -d backend` 或 `mvn spring-boot:run`）。
- [ ] 前端服务已启动（`npm run dev`），浏览器已打开 `http://localhost:5173`。
- [ ] 准备 **2 个浏览器配置文件**（或 1 主 + 1 隐身），分别登录经理/员工账号。
- [ ] 准备 **演示数据集**：
  - [ ] 至少 1 个普通员工 UserID（如 `lihua`）
  - [ ] 至少 4 个销售岗位员工 UserID（用于关系图谱）
  - [ ] 至少 1 个任务样本（用于演示双重确认）
- [ ] 企微 App 已在手机上登录（用于演示企微消息推送）。
- [ ] 网络稳定，企业微信回调地址已配置为公网可访问（如果演示远程）。
- [ ] 备用截图：万一服务挂了，直接用 README 里的截图继续讲。
