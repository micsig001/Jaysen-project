企业级任务协同管理系统 - 综合说明书
1. 项目概述
本系统是一套专为企业内部打造的任务数字化流转与效能管理平台，深度集成企业微信生态。系统以“移动端优先、PC端管理”为设计理念，旨在解决企业内部任务分派难、进度追踪弱、工时统计不准以及协作关系不透明等痛点。通过引入严格的“双重确认机制”、“权限三级隔离”及“企微消息强触达”，实现任务从创建到验收的闭环管理，同时提供全景可视化图谱，赋能管理层进行科学的资源调度与效能复盘。
2. 核心业务逻辑规范（管理基石）
任务生命周期： 已创建  →  待接收  →  进行中  →  待验收  →  已完成 。支持在“待验收”状态下驳回重做，或在“待接收”状态下撤回。
时间计算防甩锅机制：任务的倒计时与超时预警，仅在接收方点击“确认接收”时启动。系统根据指派者设定的预估时长自动推算真实截止时间，杜绝了因员工未及时查看消息导致的无效超时。
双重确认闭环：执行者完成任务后仅能提交至“待验收”状态，必须由任务发起方（或自己发给自己时的本人）点击“确认完成”才算真正完结，确保交付质量受控。
企微同步与防覆盖策略：每日凌晨全量同步组织架构，每小时增量更新。对于角色分配，若管理员在后台手动修改过用户角色（标记为  MANUAL ），企微日常同步将不会覆盖该设置，保障管理干预的有效性。
3. 界面呈现与交互设计 (UI/UX)
整体布局：采用响应式左侧导航栏 + 右侧工作区设计。左侧仿照企微消息列表风格，展示成员头像、姓名及最近动态；点击成员即可无缝切换右侧视图。
个人工作台（移动端适配）：列表页默认按优先级降序、截止时间升序排列。针对手机端操作优化，复杂操作（如驳回、确认完成）采用底部 Action Sheet（动作面板）弹出，按钮尺寸适配手指触控。超时且未完成的任务行会高亮标红并显示“已超时”标识。
任务详情页：清晰展示任务标题、来源备注、时间节点及完整状态流转记录。页面内嵌评论区，支持双方在执行过程中随时留言反馈，驳回时必须强制填写原因。
数据脱敏展示：涉及人员联系方式时，列表页默认展示脱敏格式（如  138****1234 ），仅本人或超级管理员可通过特定交互查看明文。
可视化任务流总览：
单人辐射图：以选中人员为中心节点，周围环绕直接关联人员，箭头由发起人指向接收人，直观展示个人的微观协作网络。
多人全景图（PC端专属）：选定人员在画布上形成闭合圆环或力导向布局。兜底机制：单次渲染上限为50人，超出部分默认按“任务交互频繁度”排序截取，并提供UI勾选框允许管理者手动剔除无关人员。
4. 多级权限与安全体系
普通员工：仅可见自己的任务列表及以自己为中心的辐射图。
部门经理（二级权限）：可查看本部门所有成员的任务详情，仅限查看本部门内的多人协作全景图，无法跨部门筛选。
老板/超级管理员（一级权限）：上帝视角，可跨部门组合生成全景图，拥有最高级别的数据访问权。
全局审计追踪：后端通过 AOP 拦截器对所有敏感写操作（增删改、撤回、角色变更、查看明文手机号等）进行自动日志记录，留存操作人、时间及数据快照，以备追溯。
5. 部署架构与技术栈说明
技术选型：前端 Vue 3 + TypeScript + Element Plus + ECharts；后端 Java Spring Boot + MyBatis-Plus + Redis；数据库 MySQL。
容器化部署：项目提供完整的  docker-compose.yml  编排文件，一键拉起 MySQL、Redis、Backend 和 Frontend(Nginx) 四大服务。
免登认证：作为 H5 应用集成至企微工作台，通过 OAuth2.0 授权码换取 JWT Token 实现静默登录，无需二次输入账号密码。
消息推送引擎：内置  WeWorkMessageService ，在任务分派、驳回/待验收、即将超时前1小时三个关键节点，通过指数退避重试机制向员工手机推送企微应用消息。
6. 历史归档与运维策略
冷热数据分离：独立的历史记录归档查询页面保留已完结任务供查阅。
安全归档机制：每月定时触发归档任务，严格校验“创建时间超1年”且“状态为已完成/已撤回”的双重条件，将冷数据迁移至独立的  tasks_history_archive  表后再从主表物理删除，彻底避免大批量 DELETE 导致的线上锁表瘫痪问题。
